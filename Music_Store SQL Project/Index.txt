# Music Store Database SQL Project

## Project Overview
This project involves analyzing a music store database using SQL queries to gain insights into various aspects of the store's operations, customers, and sales. The project is divided into three levels of difficulty, with questions ranging from basic to advanced queries.

## Files
- **MusicStore_database.sql**: SQL file to set up and populate the database.
- **Music_Store_SQL_Project.txt**: Contains the list of questions to be answered with SQL queries.

## Setup Instructions

1. **Clone the Repository**:
    ```bash
    git clone https://github.com/yourusername/music-store-sql-project.git
    cd music-store-sql-project
    ```

2. **Load the Database**:
   Open your preferred SQL editor and execute `MusicStore_database.sql` to create and populate the database with sample data.

## Question Sets

### Question Set 1 - Easy
1. Find the senior-most employee based on job title.
2. Identify countries with the most invoices.
3. List the top 3 invoices by total value.
4. Determine the city with the highest invoice total to plan a promotional event.
5. Identify the best customer based on total spend.

### Question Set 2 - Moderate
1. Retrieve the details of all Rock music listeners, sorted by email.
2. List the top 10 artists who have contributed the most to rock music in terms of track count.
3. Find songs longer than the average song length and list them in descending order of length.

### Question Set 3 - Advanced
1. Calculate the total amount each customer spent on each artist.
2. Identify the most popular music genre in each country based on purchase count.
3. Determine the top-spending customer for each country.

## Usage
To answer each question, run the corresponding SQL query on your database. Sample queries for each question can be written and tested within the SQL editor connected to your database.
